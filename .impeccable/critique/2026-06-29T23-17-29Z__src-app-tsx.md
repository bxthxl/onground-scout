## Anti-Patterns Verdict

**Does this look AI-generated? Yes  -  and arguably more confidently than before.** The redesign swapped one slop dialect (generic SaaS card grid) for a louder one (GSAP-showcase template + codex CSS-decoration). Lateral on quality, **worse on brand-fit**, because the new decoration actively fights "trust-first, never flashy."

New tells (all in App.css, none catchable by the static detector):
- **`.inline-photo` (App.css:104-116)**  -  a `repeating-linear-gradient` stripe blob (a banned pattern) with a 3px white border, dropped INSIDE the h1 between "rent" and "from abroad" (App.tsx:239). Reads as a broken/unloaded image wedged into the headline. `aria-hidden` confirms zero meaning. The single most damaging tell.
- **`.room-window` (App.css:203-230)**  -  a CSS-drawn window scene (mullion via gradient slivers, a radial "glint", a navy "sill"). A hand-drawn depiction of a tangible object → reads as a placeholder diagram, not premium imagery.
- **`.hero-media-panel::after` (191-201)**  -  masked 44px grid overlay; generic dark-tech dashboard texture on a brand explicitly NOT dark-tech.
- **`.checklist-preview` (359-388)**  -  a fake CSS "screenshot" of a checklist where a real screenshot belongs (aria-hidden).
- **`.marquee-track` (577-599)**  -  an infinite-scrolling adjective marquee ("Reliable / Trustworthy / On the ground …"). Decorative filler and actively anti-trust: self-applied praise sliding past in 3.8rem bold is the fly-by-night signal a scammed renter pattern-matches.
- **Still zero photographic content**  -  every place imagery is implied (scout faces, the property, the report) is a CSS gradient panel. The brand reference treats this as a bug, not restraint.
- **Over-rounding worse**  -  --radius 18px default plus 22/28/34px one-offs vs the 16px card ceiling.

**Deterministic scan:** `detect.mjs --json` on `src/App.tsx`, `src/` (which DID traverse App.css), and `src/App.css` → all `[]`, exit 0. **This is a coverage limitation, not a clean bill.** Assessment B traced the root cause: static `.css`/`.tsx` source is processed only by the regex engine, which has no matcher for decorative `repeating-linear-gradient` stripes, standalone large border-radius, or the ghost-card border+shadow combo. A `repeating-stripes-gradient` rule exists but only on the DOM/browser-engine path (live URL), which couldn't run (no browser tool exposed; dev server confirmed live at HTTP 200). So the banned patterns that demonstrably exist in App.css are structurally invisible to a static scan  -  the design review carries them.

## Overall Impression

The redesign spent real craft (GSAP scrub, pinned column, bento, marquee) but spent it on banned categories, and left the two trust-critical P0s from last round unfixed. The page now *performs at* a scared user instead of reassuring them: a striped blob breaks the headline, a section literally called "Proof" contains no proof, a marquee chants adjectives, and the journey ends on "the owner can review it from Supabase." Net score barely moved (26 → 27). Biggest opportunity unchanged: replace decoration with real evidence, and make the bottom of the page feel safe.

## What's Working

1. **Eyebrows eliminated.** The tracked-uppercase eyebrow-on-every-section anti-reference is gone; sections lead with real headlines. Concrete improvement over last round.
2. **Hero headline + subcopy stayed on-brand.** "Your eyes on the ground before you rent from abroad" + the plain feature list is specific, calm, promise-based  -  the right voice.
3. **Form status accessibility is correctly built.** role="alert" on error vs role="status" on success, icon+color+text pairing, proper autoComplete tokens, labels wrapping inputs. Real, non-trivial correctness.

## Priority Issues

**[P0] System/dev jargon in user copy, including the success message.** "Supabase environment values" / "stored in Supabase…" / "the owner can review it from Supabase" / "let the owner follow up" (App.tsx:210-211, 391, 393). For ESL, scam-fearing renters this reads as a mistake or a sketchy setup  -  fatal at the peak-end success moment. **Fix:** Success → "Thank you, [name]. We've received your request  -  a scout coordinator replies within 24 hours by email and WhatsApp." Strip all Supabase/environment/owner terms; never surface raw error.message. → **`clarify`** (+ **`harden`** for the error path). *Unfixed from last round.*

**[P0] Broken WhatsApp link with no recipient.** `https://wa.me/?text=…` (App.tsx:397) has no phone number  -  opens a dead contact picker. This is the explicit low-commitment trust on-ramp, silently failing for the exact hesitant user it's meant to catch. **Fix:** hard-code the business E.164 number. → **`harden`**. *Unfixed from last round.*

**[P1] Zero photographic evidence; CSS panels where imagery belongs.** `.inline-photo`, `.room-window`, `.checklist-preview`, `.hero-media-panel` grid all stand in for imagery the brief implies. "Show the work, don't just claim it" is the #1 principle; a trust brand with no scout faces, no real property/report photo, no testimonial cannot earn a $150 - 250 commitment from a burned buyer. **Fix:** real/stock photography  -  a scout at a building, a sample-report spread, a coordinator portrait; delete the inline-photo blob and CSS window outright. → **`colorize`** (imagery direction) + **`distill`** (remove fake-image decoration).

**[P1] Overdesigned motion fights "calm, never flashy."** Pinned 1200px desire-section (App.tsx:163-169), word-by-word opacity scrub (146-161), scrubbed contrast/saturate filters (127-144), infinite adjective marquee (374-387). "Loud/flashy/overdesigned, aggressive motion" is an explicit anti-reference; scroll-jacking and self-praising marquees raise anxiety in the target user. Motion IS technically guarded (useGSAP early-returns under reduced-motion; CSS zeroes the marquee animation) but tonally wrong. **Fix:** remove the pin and marquee; reduce word-reveal to one gentle fade; keep filters subtle and non-scrubbed. → **`quieter`**.

**[P2] Packages are decision-noise; contrast failures on labels/pills.** Basic vs Premium identical except "2" vs "5 visits" (App.tsx:55-75); form offers "Add-on only" (463) with no matching card. White on --gold price pill ≈ 2.0:1 (fails AA); gold-deep 0.78rem uppercase labels on near-white ≈ 4.0:1 (borderline-fail small text); unstyled placeholders ~2.5:1. **Fix:** differentiate package value in plain language; price-pill text to --ink (or darken pill); bump small gold labels to ≥4.5:1; set an explicit placeholder color. → **`clarify`** (packages) + **`audit`** (contrast).
